## Story

**As a** video production operator,
**I want** VideoFlow to manage briefing, DA vector, agent pipeline, concept bank, quality rubric, cost calculation, package export, and FBR-Social handoff,
**so that** authorial video production can be orchestrated with traceable human approvals.

## Acceptance Criteria

1. Visual design implements VideoFlow's distinct light theme, Instrument fonts, hero, 12-page SPA, agent stack, DA parameter groups, syntax highlighting, quality rubric, and skeleton example.
2. Data model expands DA Vector to 6 groups with 30+ params and adds HandoffSchema, QualityRubric, TemplatePreset, and AuthorialSkill.
3. Components include DA Vector editor, production chain view, concept bank, cost calculator, and template presets.
4. Database, API routes, BullMQ worker, pgvector concept bank, SSE/progress, package generation, and human intervention match `PRD_SPEC_TASKLIST.md`.
5. Arva integration supports 6 slots: Orquestrador, Referencias, Elementos, Editor, Publicacao, Supervisor.
6. FBR-Social receives the publication package integration.
7. Tests cover DA vector persistence, handoff immutability, pgvector search, worker pipeline, cost calculation, package ZIP, human intervention, and UI flows.

#### Blocking Issues

1. Story scope is too large for a ready implementation story. It aggregates `VF-V01` through `VF-V09`, `VF-D01` through `VF-D02`, `VF-C01` through `VF-C05`, `VF-A01`, and `TASK-VF01` through `TASK-VF18`; the source docs already separate this work across multiple ordered tasks and four sprints.
2. Dev Notes are not self-contained enough for Article IV. They cite source files but do not summarize the concrete DA vector schema, Handoff envelope rules, database schema/RLS, API contracts, SSE event format, BullMQ queue names, FBR-Social payload/auth, storage paths, rate limits, retry rules, or exact dependency sequence needed for implementation.
3. File locations are incomplete for the requested blast radius. The story names broad path groups such as `apps/modules/videoflow`, `backend/api/videoflow`, workers, migrations, and package services, while the repo currently has `apps/portal` and the source tasklist names concrete `src/app/api/proxy/videoflow/...`, `src/lib/videoflow/...`, `src/components/videoflow/...`, and `src/workers/...` files.
4. Acceptance criteria are testable only after external decomposition. AC 4 and AC 7 require matching the entire `PRD_SPEC_TASKLIST.md` and broad test classes without embedding the specific pass/fail conditions in the story.
