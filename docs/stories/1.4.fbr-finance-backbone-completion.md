## Story

**As a** finance manager,
**I want** FBR-Finance to centralize receivables, payables, cost centers, reconciliation, forecasting, audit, reports, and agent operations,
**so that** financial activity across the group is traceable, governed, and reconciled.

## Acceptance Criteria

1. Visual design implements Finance CSS variables, hero KPIs, 6 module cards, 7 agent cards, integration table, and governance section.
2. Database model includes the 14 tables listed in `TASKS_FINANCE.md`, with company isolation, RLS, approval flows, audit log, and agent actions.
3. Receivables, payables, cost centers, P&L, reconciliation, forecast, audit, reports, and anomaly alert routes match `PRD_SPEC_TASKLIST.md`.
4. Frontend includes dashboard, receivables table, payments/approval workflow, cost center tree, P&L, reconciliation queue, forecast graph, payment calendar, audit log, reports, and anomaly alerts.
5. Arva integration supports 7 finance slots: Vault, Intake, Payout, Ledger, Reconcile, Oracle, Sentinel.
6. Cross-module financial events and Sales payment events can create finance records as specified.
7. Tests cover migrations, RLS, approval rules, reconciliation scoring, forecast calculation, audit immutability, report exports, and UI critical flows.

- All group systems generate financial events that arrive in FBR-Finance, including FBR-Sales partner payments, FBR-Click contract values, and infra/license/tooling costs from any module. [Source: `fbr-portal-docs/03_FINANCE/PRD_SPEC_TASKLIST.md#2-posicao-no-ecossistema`]

### Cross-Module Events

- `payment.received` from FBR-Sales carries partner id/name, company id, amount, currency, reference period, receipt date, partnership type, optional receipt path, and notes; Finance creates a pending receivable and attempts automatic reconciliation. [Source: `fbr-portal-docs/00_GERAL/DOCUMENTO_GERAL.md#113-evento-paymentreceived-fbr-sales-fbr-finance`]
- `financial.event` from any module carries cost type, description, amount, company id, optional cost center, recurrence flag, reference date, and source module; Finance registers it as a pending payable linked to the correct cost center. [Source: `fbr-portal-docs/00_GERAL/DOCUMENTO_GERAL.md#117-evento-financialevent-qualquer-modulo-fbr-finance`]
- Cross-module communication must use REST event payloads through module APIs; modules must not write directly to another module database. [Source: `fbr-portal-docs/00_GERAL/DOCUMENTO_GERAL.md#33-comunicacao-entre-modulos`]
