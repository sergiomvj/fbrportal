# Story 3.6.2: FBR-Social Render Pipeline, Gallery, ZIP, and Quality Checks

## Status

Complete ✅ (2026-05-14)

## Story

**As a** social production lead,  
**I want** briefs to generate rendered packages with manifest and quality checks,  
**so that** Social can deliver production-ready assets.

## Acceptance Criteria

1. Brief/job queue persists input, status, target formats, and generated outputs.
2. Render pipeline records per-format output metadata.
3. Gallery lists deliverables from persisted artifact data.
4. ZIP package includes generated files and a valid manifest.
5. Naming convention and folder structure match the PRD/SPEC.
6. Quality checks cover dimensions, safe zone, size, colors, logo, and final checklist.
7. Tests cover render state, gallery, ZIP manifest, naming, and quality failures.
