## Story

**As a** social production owner,  
**I want** FBR-Social reviewed and corrected against the original Social Media Visual Agent concept,  
**so that** artwork generation, template governance, quality control, packaging, and brand-kit synchronization fully match the intended product.

## Acceptance Criteria

1. The implemented Social scope is reviewed against `fbr-portal-docs/08_SOCIAL/PRD_SPEC_TASKLIST.md`, `fbr-portal-docs/08_SOCIAL/TASKS_SOCIAL.md`, and `fbr-portal-docs/08_SOCIAL/concept.html`, with all shipped capabilities classified as conformant, partial, or missing.
2. Core Social functionality matches the original concept: network/format catalog, job creation, rendering pipeline, safe zones, quality checks, gallery, template versioning, package manifest, and ZIP delivery.
3. Information flow matches the original ecosystem design: brand-kit data enters from Design through the documented webhook/event path, social render/package data stays inside Social, and any downstream outputs preserve proxy-only boundaries.
4. Human/agent responsibilities, quality gates, template governance, and audit/security behavior match the original docs and the portal-wide general architecture.
5. All identified Social gaps produce concrete code and test changes without adding new visual-production scope outside the original docs.
6. Existing Social story `1.9.fbr-social-visual-production-completion.md` is revalidated against the original concept, and any residual drift is explicitly closed.
7. Quality gates pass for the final conformity increment: `npm run lint`, `npm run typecheck`, `npm test`, and `npm run build`.
