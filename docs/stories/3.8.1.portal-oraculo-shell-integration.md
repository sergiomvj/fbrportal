# Story 3.8.1: Portal UI Oraculo Shell Integration

## Status

Complete ✅ (2026-05-14)

## Story

**As a** portal user,  
**I want** Oraculo accessible from the main portal shell,  
**so that** the already functional Oraculo module is part of the product navigation and session experience.

## Acceptance Criteria

1. Main navigation exposes a clear Oraculo entrypoint.
2. Oraculo shell is visually consistent with the portal modules.
3. Loading, empty, error, and session states are implemented.
4. Oraculo uses the portal session/context boundary.
5. Notifications and responsive layout are validated.
6. Tests cover navigation, route access, states, and session behavior.
