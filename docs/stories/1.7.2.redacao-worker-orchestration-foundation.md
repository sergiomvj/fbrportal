## Story

**As a** newsroom operator,  
**I want** Redacao worker orchestration represented by real queues, retries, and workflow automation,  
**so that** article processing can move beyond a mocked portal simulation.

## Acceptance Criteria

1. The story records the current state accurately: queue names and agent statuses are present only as mocked metadata in the portal store.
2. No dedicated `Celery`, `n8n`, retry-handler, or SSE worker implementation is falsely marked as complete.
3. The remaining work for real orchestration is explicit and implementation-ready.

## Tasks / Subtasks

- [x] Record the current mocked orchestration surface. (AC: 1)
  - [x] Queue names such as `redacao.monitor` and `redacao.publisher` are represented in `store.ts`.
  - [x] Agent status is surfaced through dashboard data and UI cards.
- [x] Remove incorrect completion claims for worker architecture. (AC: 2)
