## Story

**As a** marketing strategy owner,  
**I want** FBR-MKT rebuilt or corrected against the original Marketing Intelligence concept,  
**so that** uploaded viability documents become diagnostics, strategy, copy, calendar, roadmap, chat, and exports exactly as the product was originally designed.

## Acceptance Criteria

1. The implemented MKT scope is reviewed against `fbr-portal-docs/05_MKT/PRD_SPEC_TASKLIST.md`, `fbr-portal-docs/05_MKT/TASKS_MKT.md`, and `fbr-portal-docs/05_MKT/concept.html`, with all shipped capabilities classified as conformant, partial, or missing.
2. Core MKT functionality matches the original concept: document upload (PDF/DOCX), diagnostic extraction (SWOT, persona, UVP, score), strategy master, copywriting, 90-day editorial calendar, roadmap, contextual chat, and PDF/PPTX export.
3. Information flow matches the original ecosystem design: asynchronous document processing, strategy version snapshots, contextual chat scoped to the active strategy, and `strategy.exported` delivery toward FBR-Click without duplicating versions or bypassing the proxy/backend boundary.
4. Agent workflow matches the original concept: Extrator, Estrategista, Redator, Calendario, Exportador, and Onboarding operate in the correct order with observable status and immutable outputs where documented.
5. The implementation no longer uses a generic campaigns-only domain model where the original product requires strategy-generation workflow and diagnostic/version/chat/export entities.
6. All identified MKT gaps produce concrete code, migration, route, UI, worker, and test changes without inventing new product scope outside the original docs.
7. Quality gates pass for the final conformity increment: `npm run lint`, `npm run typecheck`, `npm test`, and `npm run build`.
