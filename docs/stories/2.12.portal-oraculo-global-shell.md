## Story

**As a** FBR Portal operator,
**I want** the Oraculo available on every portal screen with contextual suggestions,
**so that** I can inspect how the system works without leaving the current module.

## Acceptance Criteria

1. The portal root layout mounts a shared Oraculo shell for every authenticated route except `/login`, with open/close state and panel width persisted in `localStorage`. [Source: `fbr-portal-docs/Oraculo/prd.md#41-painel-lateral-fixo-ide-style`]
2. The Oraculo opens via button and `Ctrl+K` / `Cmd+K`, occupies the right side of the portal without overlaying the main content, and supports drag resize from a default width of 320px. [Source: `fbr-portal-docs/Oraculo/prd.md#41-painel-lateral-fixo-ide-style`]
3. The shell derives current module and screen context from the active route, emits `ORACULO_CONTEXT` through Portal Bridge semantics, and shows contextual suggested questions per screen. [Source: `fbr-portal-docs/Oraculo/prd.md#44-contexto-automatico-de-tela`, `fbr-portal-docs/Oraculo/prd.md#45-perguntas-sugeridas-por-tela`]
4. `POST /api/oraculo/query` only returns answers grounded in real repository files mapped to the active module or shared portal docs, with file and line citations for every source shown in the UI. [Source: `fbr-portal-docs/Oraculo/prd.md#42-interpretacao-de-codigo-capacidade-central`, `fbr-portal-docs/Oraculo/prd.md#43-rag-sobre-documentacao`]
5. The initial portal delivery does not execute code, mutate system data, or fabricate unsupported answers; when evidence is insufficient, the API returns an explicit limitation message. [Source: `fbr-portal-docs/Oraculo/prd.md#6-o-que-o-oraculo-nao-faz`]
6. Quality gates pass: `npm run lint`, `npm run typecheck`, and `npm test`. [Source: `AGENTS.md#quality-gates`]

## Dev Notes

### Scope Decision

This story implements the PRD recorte approved by the user for "option 1": shared portal UX plus an initial portal-native query backend. It explicitly excludes embeddings, pgvector, Supabase history, webhook reindexing, and external LLM orchestration from Sprints 1 and 3 of the PRD.

### Invariants

- The Oraculo must never claim behavior without citing a real file and line.
- The initial query route must stay read-only over the repository and portal docs.
- The shared shell must live in the root portal layout, not duplicated across modules.
- Login remains outside the Oraculo shell.
