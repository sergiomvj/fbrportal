## Story

**As a** finance manager,
**I want** asynchronous reconciliation with score bands and a human review queue,
**so that** bank statement matching can be automated while preserving review control for uncertain cases.

## Acceptance Criteria

1. Asynchronous reconciliation job processing and progress tracking exist for queue `conciliacao`.
2. Matching score supports 0-100 and decision bands: score >= 80 auto-reconciles, score 50-79 goes to human review, score < 50 remains unreconciled.
3. Large statements are processed in batches of 100.
4. Duplicate statements are rejected, already reconciled items are skipped, Redis/job retry behavior is handled, and jobs exceeding 5 minutes enter failure state.
5. APIs exist for `POST /api/proxy/finance/conciliacao/run`, `GET /api/proxy/finance/conciliacao/pendencias`, status polling, and pending-item approve/reject/create-entry actions.
6. Reconciliation page includes run button, progress, pending table/cards, score colors, side panel details, approve/reject/create actions, retry/error state, and empty state.
