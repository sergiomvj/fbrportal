## Story

**As a** pre-sales operator,
**I want** FBR-Click to manage deals, messages, tasks, agents, KPIs, audit logs, and FBR-Leads handoff,
**so that** qualified leads can move reliably from first contact to first sale.

## Acceptance Criteria

1. Visual completion covers Click CSS variables, hero, KPI bar, pipeline, agent cards, message bubbles, architecture diagram, and security grid.
2. Data model covers all 13 Click entities listed in `TASKS_CLICK.md`, with TypeScript types for the entities and related microservice/security/LLM concepts.
3. Components include redesigned deal cards, create deal modal, deal detail, timeline, messages panel, and agent dashboard.
4. Arva integration adds "Incluir Agente" filtered by commercial/sales tags and supports the 6 Click agent slots.
5. API completion includes `/api/proxy/click/deals/from-lead`, agent trigger/status routes, audit log, and audit CSV export.
6. FBR-Leads `lead.qualified` can create a Click deal in `contato_inicial`.
7. Tests cover deal creation, handoff from Leads, stage movement, messages, task status, agent controls, audit export, and RLS/security expectations.

The main risks are functional gaps in the create-deal UI and agent controls, plus audit semantics that can produce misleading history for FBR-Leads handoff.
