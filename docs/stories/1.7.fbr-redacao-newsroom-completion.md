## Dev Notes

### Current Repo Baseline

- The Redacao module already exists in the current repo shape:
  - `apps/portal/src/app/redacao/*`
  - `apps/portal/src/app/api/proxy/redacao/*`
  - `apps/portal/src/lib/redacao/*`
- The current implementation is a portal-scoped prototype backed by an in-memory store and Zod schemas, not by persistent backend services.

### Source-Doc Gaps Still Open

- The source docs still expect dedicated backend and orchestration layers such as:
  - `backend/api/redacao/*`
  - `backend/workers/redacao/*`
  - `backend/services/wordpress.py`
  - `n8n/workflows/*`
- Those paths are not implemented in this repo. Their responsibilities are only partially represented in the current portal/store prototype.

### Conformity Summary

- Already present in code:
  - Overview, production, published, sources, UGC, alerts, and agents pages.
  - Proxy endpoints for artigos, dashboard, fontes, alertas, agentes, and UGC.
  - In-memory multi-tenant filtering by `company_id`.
  - Editorial stage metadata, alert taxonomy, and mocked queue names/statuses.
- Still missing versus source docs:
  - Persistent database and migrations.
  - Real RLS.
  - Dedicated workers and retries.
  - n8n workflow exports.
  - WordPress publication.
  - SSE / WebSocket live stream.
  - Downstream `article.published` integration.
  - Redacao-focused automated tests.
