### Data Model Requirements

- `artigos`: UUID primary key, `empresa_id`, `titulo`, `conteudo_pt`, `conteudo_en`, `cidade`, `tipo` (`noticia`, `analise`, `traducao`), `fonte_url`, `fonte_nome`, `etapa` default `coletado`, `agente_atual`, `url_publicado`, `imagem_url`, timestamps, and publication timestamp.
- `fontes_rss`: UUID primary key, `cidade`, `url`, `nome`, `ativo` default true, `ultimo_ok`, and creation timestamp.
- `ugc_pautas`: UUID primary key, `submissor`, `email`, `cidade`, `descricao`, `status` default `pendente`, `artigo_id`, creation timestamp, `auto_aprovavel` default false, and `score_confianca` default 0.
- `ugc_creditos`: UUID primary key, `submissor`, `email`, `creditos` default 0, and `updated_at`.
- `alertas_redacao`: UUID primary key, `tipo`, `mensagem`, `nivel` default `info`, `resolvido` default false, optional `artigo_id`, `agente`, JSONB `contexto`, `resolvido_por`, `resolvido_em`, and creation timestamp.
- RLS requirement from TASK-RD01: each company only sees its own articles via `empresa_id`.
- Index requirements include article stage/city/company indexes, RSS city index, UGC status index, alert severity/type indexes, and a GIN index for article full-text search.

### Alert Classification

| Tipo | Nivel padrao | Automatic action |
|---|---|---|
| `falha_agente` | `error` | Notify operator via SSE and move article to dead letter. |
| `fonte_indisponivel` | `warn` | Pause source automatically and notify operator. |
| `conteudo_sensivel` | `warn` | Block automatic publication and require human review. |
| `queda_performance` | `warn` | Notify operator and suggest checking sources. |
| `qualidade_baixa` | `info` | Requeue Editor review. |
| `imagem_nao_encontrada` | `info` | Use placeholder image and mark article for media review. |
| `limite_api_atingido` | `error` | Pause calls for 1 hour and schedule automatic retry. |
| `ugc_spam_detectado` | `info` | Mark UGC submission as rejected and do not notify operator. |

Auto-resolution rules:
- `fonte_indisponivel`: auto-resolved when the feed returns HTTP 200 OK again.
- `limite_api_atingido`: auto-resolved after 1 hour.
- `falha_agente`: auto-resolved when the article is processed successfully in retry.

### UGC / Eu Reporter Rules

- Public submission payload: `{ submissor, email, cidade, descricao }`.
- Initial validation: description has at least 20 characters, email is valid, city is covered, and spam filters run against forbidden words/suspicious links.
- Confidence score is 0-100: +30 registered email, +20 covered city match, +25 description over 100 characters and coherent, +15 prior accepted submissions, -40 external links, -30 spam words.
- Score `>= 60` sets `auto_aprovavel=true`; lower scores remain pending manual review with `auto_aprovavel=false`.
- Accepting UGC creates an article with `etapa='coletado'` and triggers the editorial pipeline.
- Credits: +10 when a pauta is accepted, +20 when the derived article is published, and every 100 credits gives highlight on the "Reporteres da Comunidade" page.

### Full-Text Search

- PostgreSQL text search configuration: `CREATE TEXT SEARCH CONFIGURATION pt_br (COPY = portuguese)`.
- Mapping must use `unaccent` and `portuguese_stem` for `hword`, `hword_part`, and `word`.
- Article search index must use `to_tsvector('pt_br', coalesce(titulo, '') || ' ' || coalesce(conteudo_pt, ''))`.
- Search endpoint from source docs: `GET /api/proxy/redacao/busca?q=...&cidade=...`; results rank by `ts_rank`, filter by city when supplied, order by rank then `created_at`, and limit to 20.

### Celery Worker Architecture

| Queue | Agent | Concurrency | Retry policy |
|---|---|---:|---|
| `redacao.monitor` | Monitor | 4 | exponential backoff, max 5 retries, base delay 60s |
| `redacao.jornalista` | Jornalista | 2 | exponential backoff, max 3 retries, base delay 120s |
| `redacao.midia` | Midia | 3 | exponential backoff, max 3 retries, base delay 90s |
| `redacao.editor` | Editor | 2 | exponential backoff, max 3 retries, base delay 120s |
| `redacao.publisher` | Publisher | 2 | exponential backoff, max 5 retries, base delay 60s |
| `redacao.gestor` | Gestor | 1 | exponential backoff, max 3 retries, base delay 180s |

- Celery config requirements: route tasks by queue, `task_acks_late=True`, `worker_prefetch_multiplier=1`, `task_soft_time_limit=300`, and `task_time_limit=600`.
- Editorial chain: Jornalista -> Midia -> Editor -> Publisher.
- Retry handler: increment `retry_count`, retry below 3 attempts with delays 60s/120s/240s, mark article `etapa='erro'` and create `falha_agente` alert at 3+ attempts.
- Dead letter queue: `redacao.dead_letter` with 7-day TTL, per TASK-RD13.

### n8n Workflow Requirements

- `01_monitor_coleta_rss.json`: cron every configurable interval, read active RSS sources by city, fetch/parse feeds, deduplicate by `fonte_url`, apply exclusion words, insert new articles as `coletado`, retry HTTP errors 3 times with 60s delay, and create `fonte_indisponivel` after 3+ city feed failures.
- `02_monitor_deduplicacao.json`: source-doc companion workflow for monitor deduplication.
- `03_pipeline_editorial.json`: trigger when article reaches `coletado`; Jornalista creates PT-BR `conteudo_pt`/`titulo`, Midia fetches licensed image, Editor approves/rejects, Publisher posts to WordPress, and each step emits SSE updates.
- `04_retry_falha.json`: receives `{ artigo_id, agente, erro }`, increments retry count, reschedules or marks `erro`, and creates `falha_agente`.

### WordPress Integration

- Authentication uses WordPress Application Passwords or JWT plugin.
- Environment variables named by source docs: `WP_API_URL`, `WP_USER`, `WP_APP_PASSWORD`.
- Post payload includes `title`, `content`, `status` (`publish`, `draft`, or `pending`), `categories`, optional `featured_media`, and meta fields for `cidade`, `tipo_conteudo`, and `fonte_original`.
- WordPress 5xx errors retry 3 times; if still failing, article becomes `etapa='erro'`.
