## Story

**As a** newsroom operator,
**I want** FBR-Redacao to orchestrate articles, published content, UGC moderation, alerts, RSS sources, WordPress publishing, and editorial agents,
**so that** local news production runs continuously with human control points.

## Acceptance Criteria

1. Visual design implements Redacao CSS variables, hero/KPIs, coverage grid, distribution channels, content type cards, and 6-agent pipeline.
2. Data model includes articles with Portuguese full-text search, UGC with confidence/credits, and alerts with 8 types and auto-resolution.
3. Components include production mural, published mural, agent dashboard, Eu Reporter queue, alert center, and full-text search.
4. Celery worker architecture, n8n workflows, WordPress REST integration, UGC moderation, pt-BR full-text search, and alert classification match the SPEC.
5. Arva integration supports 6 slots: Coletor, Jornalista, Arte, Editor Regional, Chefe de Redacao, Moderador.
6. `article.published` integration can update FBR-Sales audience/media kit data.
7. Tests cover RSS collection, editorial pipeline, retry logic, WordPress calls, UGC moderation, full-text search, alerts, and UI flows.

### Critical Issues

1. The story is not self-contained enough for implementation. Dev Notes mostly point to external documents instead of embedding the implementation context required by the story template.
2. Scope is too broad for one implementation-ready increment: visual design, schema/RLS, API, frontend, Celery, n8n, WordPress, UGC moderation, search, alerts, Arva, and `TASK-RD01` through `TASK-RD18` are bundled together.
3. File structure is not sufficiently validated against the current repository. The source docs name production paths such as `apps/modules/redacao`, `backend/api/redacao`, `backend/workers/redacao`, and `n8n/workflows`, but those paths are not present in the current repo.
4. Executor assignment has a framework-rule conflict: the story uses `quality_gate: "@qa"`, which aligns with Constitution Article II quality verdict authority, while `validate-next-story.md` still lists `@architect`, `@dev`, and `@pm` as allowed quality gates.

### Should Fix

- Split the story or expand subtasks so each AC has concrete file paths, dependencies, and validation steps.
- Pull the key Redacao technical details into Dev Notes: 8 alert types, auto-resolution rules, UGC scoring/credits, pt-BR search configuration, Celery queues/retry policy, n8n workflow names, and `article.published` payload.
- Clarify CLI/backend-first sequence before UI work to satisfy Constitution Article I.
- Specify actual test file locations and commands for backend workers, API routes, frontend flows, n8n boundaries, WordPress calls, UGC moderation, search, and alerts.
