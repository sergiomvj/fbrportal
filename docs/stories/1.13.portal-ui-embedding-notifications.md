## Story

**As a** FBR Portal operator,
**I want** secure module iframe embedding, module routing, role guards, realtime notifications, and notification UI,
**so that** authenticated users can access allowed modules and receive cross-module updates safely.

## Acceptance Criteria

1. `ModuleFrame` renders sandboxed iframe, loading skeleton, 10s timeout error, retry, `MODULE_READY -> AUTH_TOKEN`, and `NAVIGATION` URL sync. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P14`]
2. Nine module routes render `ModuleFrame` with role guard, unauthorized screen, `/` default redirect by role, fail-closed DB access errors, and middleware protection. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P15`]
3. Supabase Realtime notifications include migration, RLS, provider/context/hooks, read/read-all routes, deduplication, reconnect behavior, and user isolation. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P16`]
4. Notification UI includes toast stack, error-toast persistence, right drawer, module filter, read-all action, and sidebar badge updates. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P17`]
5. `postMessage` security rejects unknown origins, invalid navigation URLs, late `MODULE_READY` after timeout, and unloaded iframe errors without throwing to the user.
6. Tests cover iframe readiness/auth flow, routing guards, unauthorized access, Supabase notification RLS, read operations, toasts, drawer behavior, and sidebar badge updates.
7. Quality gates pass: lint, typecheck, tests, and build if available.
