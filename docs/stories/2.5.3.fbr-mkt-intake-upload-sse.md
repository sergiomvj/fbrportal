## Story

**As a** marketing strategist,  
**I want** to create a project from either an adaptive company-description wizard or a document upload with live processing status,  
**so that** the system can start the diagnostic workflow from real business context instead of dummy records.

## Acceptance Criteria

1. The intake experience supports the concept's adaptive wizard for company description, goals, and intended next steps, and also supports the documented file-upload alternative.
2. Upload accepts only approved document types from source artifacts and enforces size, MIME, extension, and approved file-safety checks before processing begins.
3. Uploaded or wizard-produced intake data creates a real MKT project/document context persisted in the approved storage model, including document path/metadata when file upload is used.
4. Processing progress is observable through SSE or equivalent source-approved streaming semantics with the required lifecycle states for extraction and generation flow, including reconnect-safe current-state retrieval.
5. Reconnect/current-state behavior exists so a page reload or reconnection does not lose job state.
6. The intake flow emits the source-required ready-state notification path when strategy generation becomes available, without requiring the user to poll the shell manually.
7. The current static or implied upload flow in the shell is replaced by a real intake surface with user-entered company context.
