## Story

**As a** finance manager,
**I want** receivables APIs, dashboard metrics, and Sales payment intake,
**so that** incoming revenue is visible, filterable, and automatically prepared for reconciliation.

## Acceptance Criteria

1. `GET /api/proxy/finance/recebimentos` supports filters for parceiro, status, date range, company, pagination, and company isolation.
2. `POST /api/proxy/finance/recebimentos` validates required fields, positive monetary values, valid company, and returns descriptive 400/422 errors.
3. `PATCH /api/proxy/finance/recebimentos/{id}/conciliar` marks divergence <= 5% as received, divergence > 5% as divergent, and already reconciled items return 409.
4. Dashboard API returns `receita_total`, `a_receber`, `atrasados`, `divergencias`, `projecao_30d`, previous-period delta, revenue by partner, and revenue by company.
5. Sales `payment.received` intake creates pending receivables and can trigger automatic reconciliation.
6. Frontend includes dashboard KPI cards and a receivables table with filters, create modal, validation, badges, sorting, pagination, loading, retry, and empty states.
