## Story

**As a** newsroom operator,
**I want** the Redacao article, RSS, UGC, credit, alert, and search data foundation implemented with tenant isolation,
**so that** newsroom automation can persist and query production data before UI or worker orchestration starts.

## Acceptance Criteria

1. Data model covers `artigos`, `fontes_rss`, `ugc_pautas`, `ugc_creditos`, and `alertas_redacao` with the fields, defaults, timestamps, JSON context, and relationships defined in parent story 1.7.
2. `artigos` is company-scoped by `empresa_id` and has RLS coverage so each company only sees its own articles.
3. Indexes cover article stage/city/company, RSS city, UGC status, alert severity/type, and article full-text search.
4. PostgreSQL pt-BR full-text search uses a `pt_br` configuration copied from `portuguese`, with `unaccent` and `portuguese_stem` mapped for `hword`, `hword_part`, and `word`.
5. Migration rollback coverage exists for tables, indexes, RLS policies, and text-search configuration.
6. Repository-level gates pass: `npm run lint`, `npm run typecheck`, and `npm test`.
