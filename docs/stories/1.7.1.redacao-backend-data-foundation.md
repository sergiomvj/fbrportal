## Story

**As a** newsroom operator,  
**I want** a reliable Redacao data foundation for articles, sources, UGC, alerts, and dashboard queries,  
**so that** the portal can expose editorial state consistently before the full backend architecture lands.

## Acceptance Criteria

1. The current repo exposes Redacao domain schemas for articles, sources, UGC, alerts, agents, and dashboard KPIs.
2. The current repo exposes proxy routes backed by a shared Redacao store for articles, dashboard, sources, alerts, agents, and UGC.
3. The story explicitly records that the current implementation is in-memory and company-filtered, not persistent DB + RLS.
4. Missing source-doc requirements such as migrations, persistent storage, and real RLS are captured as open work instead of being marked complete.

## Tasks / Subtasks

- [x] Implement portal-scoped Redacao schemas. (AC: 1)
- [x] Implement portal-scoped Redacao store functions for listing and mutating core newsroom entities. (AC: 1, 2)
- [x] Expose proxy endpoints for the current store-backed flows. (AC: 2)
- [x] Document current implementation constraints honestly. (AC: 3, 4)
- [ ] Add persistent database, migrations, and real RLS parity with the source docs. (AC: 4)
- [ ] Add automated Redacao backend tests that cover these routes and data rules. (AC: 4)

## Dev Notes

### What Exists Now

- `apps/portal/src/lib/redacao/types.ts`
  - Defines Zod schemas for `Artigo`, `FonteRSS`, `UGCSubmission`, `Alerta`, `RedacaoAgent`, and `DashboardKpis`.
- `apps/portal/src/lib/redacao/store.ts`
  - Holds in-memory seed data and store operations for `artigos`, `fontes`, `ugc`, `alertas`, `agentes`, and dashboard KPIs.
  - Applies company filtering through `company_id` on store reads and mutations.
- Proxy endpoints:
  - `apps/portal/src/app/api/proxy/redacao/artigos/route.ts`
  - `apps/portal/src/app/api/proxy/redacao/dashboard/route.ts`
  - `apps/portal/src/app/api/proxy/redacao/fontes/route.ts`
  - `apps/portal/src/app/api/proxy/redacao/alertas/route.ts`
  - `apps/portal/src/app/api/proxy/redacao/agentes/route.ts`
  - `apps/portal/src/app/api/proxy/redacao/ugc/route.ts`

### What Does Not Exist Yet

- No Supabase / SQL migration layer for Redacao.
- No persistent tables for `artigos`, `fontes_rss`, `ugc_pautas`, `ugc_creditos`, or `alertas_redacao`.
- No real PostgreSQL `pt_br` text-search configuration.
- No real row-level security; current isolation is application-side in-memory filtering only.
