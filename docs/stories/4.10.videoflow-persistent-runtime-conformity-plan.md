# Story 4.10: FBR-VideoFlow Persistent Runtime Conformity Plan

## Status

Draft

## Story

**As a** program owner,  
**I want** VideoFlow converted from local store/prototype behavior into a dedicated persistent-runtime plan,  
**so that** the module is not accidentally marked conformant without PRD-backed system resources.

## Problem to Fix

VideoFlow is explicitly outside the current main queue, but current runtime uses in-memory productions, concepts, agents, and templates. It needs a separate conformity plan before implementation closure.

Current evidence:

- `apps/portal/src/lib/videoflow/store.ts` keeps productions, concepts, agents, and templates in arrays.
- The backlog states VideoFlow should be treated in a separate plan.

## What Must Be Implemented

1. Re-read VideoFlow PRD:
   - `fbr-portal-docs/09_VIDEOFLOW/PRD_SPEC_TASKLIST.md`;
   - any app/module reference under `fbr-portal-docs/apps/modules/videoflow`.
2. Produce a VideoFlow system-resource matrix:
   - entities;
   - persistence/RLS;
   - storage;
   - workers/BullMQ;
   - embeddings/pgvector if required;
   - integrations;
   - UI surfaces;
   - tests.
3. Create implementation stories after the plan:
   - data foundation;
   - production pipeline;
   - agents/workers;
   - storage/artifacts;
   - UI/operator surface;
   - final conformity evidence.
4. Prevent false completion:
   - update `docs/stories/2.9.videoflow-conformity-alignment.md`;
   - update `docs/stories/3.9.1.videoflow-conformity-plan.md`;
   - record that current in-memory store is not conformant.

## What Must Be Changed or Reworked

- Do not immediately rewrite VideoFlow runtime in this story unless the plan is approved.
- Do identify all files currently using `apps/portal/src/lib/videoflow/store.ts`.
- Do define the migration and worker strategy for later stories.

## Acceptance Criteria

1. VideoFlow has a PRD-backed system-resource implementation plan.
2. The plan states exactly which runtime pieces are in-memory today.
3. The plan defines required migrations, workers, storage, queues, and tests.
4. Follow-up stories exist or are listed with clear titles and dependencies.
5. No document marks VideoFlow fully conformant until implementation stories are done.

## Test Requirements

- Planning story only: no runtime tests required.
- If docs lint/check exists, run it.
- Future stories must include lint/typecheck/test gates.

## Files Likely To Change

- `docs/stories/2.9.videoflow-conformity-alignment.md`
- `docs/stories/3.9.1.videoflow-conformity-plan.md`
- new VideoFlow implementation stories under `docs/stories/`
- optionally `docs/videoflow-system-resource-plan.md`

- VideoFlow has a concrete plan and is not treated as conformant by accident.
