# Story 3.2.1: FBR-Click Deal Core and Stage Pipeline

## Status

Complete âś… (2026-05-14)

## Story

**As a** commercial operator,  
**I want** Click deals created manually and automatically with validated PRD stages,  
**so that** the CRM pipeline is reliable and auditable.

## Acceptance Criteria

1. Manual deal creation follows the Click PRD schema and company isolation.
2. Automatic deal creation from `lead.qualified` preserves full source context.
3. Pipeline stages match the official PRD stage list.
4. Stage transitions enforce valid rules and record actor, timestamp, previous stage, and next stage.
5. Deal ficha exposes required fields, source events, owner, value, tasks/messages summary, and current stage.
6. Tests cover manual creation, lead ingestion, invalid transitions, and ficha retrieval.
