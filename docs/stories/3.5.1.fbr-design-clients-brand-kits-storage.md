# Story 3.5.1: FBR-Design Clients, Brand Kits, and Storage

## Status

Complete ✅ (2026-05-14)

## Story

**As a** design operator,  
**I want** clients and brand kits persisted with logo storage and live preview,  
**so that** creative jobs can use real brand assets.

## Acceptance Criteria

1. Clients can be created, listed, updated, and scoped by company.
2. Brand kits support CRUD for colors, typography, logos, and usage rules.
3. Logo upload stores files in the approved storage path with validation.
4. Live preview renders from persisted brand kit data.
5. Brand-kit changes are auditable and usable by downstream modules.
6. Tests cover client CRUD, brand kit CRUD, logo validation/storage, and preview data.
