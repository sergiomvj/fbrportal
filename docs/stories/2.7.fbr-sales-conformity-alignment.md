## Story

**As a** revenue operations owner,  
**I want** FBR-Sales reviewed and corrected against the original digital commercial department concept,  
**so that** partner/commercial lifecycle, monetization controls, anomalies, media kits, and downstream finance signals fully match the intended product.

## Acceptance Criteria

1. The implemented Sales scope is reviewed against `fbr-portal-docs/07_SALES/PRD_SPEC_TASKLIST.md`, `fbr-portal-docs/07_SALES/TASKS_SALES.md`, and `fbr-portal-docs/07_SALES/concept.html`, with all shipped capabilities classified as conformant, partial, or missing.
2. Core Sales functionality matches the original concept: intake from closed deals, commercial/partner lifecycle, contract/onboarding activation, anomaly review, rate cards, media kits, and revenue observability.
3. Information flow matches the original ecosystem design: `deal.closed` enters from Click, commercial state evolves inside Sales, and `payment.received` or equivalent financial events flow to Finance through the documented integration contract.
4. Human/agent responsibilities, approval gates, audit evidence, and security boundaries match the original docs and the portal-wide general architecture.
5. All identified Sales gaps produce concrete code and test changes without adding new commercial scope outside the original docs.
6. Existing Sales story `1.8.fbr-sales-revenue-completion.md` is revalidated against the original concept, and any residual drift is explicitly closed.
7. Quality gates pass for the final conformity increment: `npm run lint`, `npm run typecheck`, `npm test`, and `npm run build`.
