## Story

**As a** program owner,  
**I want** the remaining PRD backlog split into executable stories,  
**so that** every pending module item has a clear path from requirement to code, tests, and evidence.

1. Every pending module from `docs/backlog-pendente-por-modulo-prds.md` is represented by one or more executable stories.
2. Each story names its source PRD/SPEC and the relevant conformity story.
3. Each story has clear acceptance criteria, tasks, test expectations, and done evidence.
4. No story claims completion for FBR-MKT unless a regression appears; MKT stays closed in the current state.
5. Execution order follows the priority queue: Leads, Click, Sales, Redacao, Design, Social, Finance, Portal UI/Oraculo, then VideoFlow.
