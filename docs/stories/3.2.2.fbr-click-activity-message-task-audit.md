# Story 3.2.2: FBR-Click Activity Surfaces, Message Hub, Tasks, and Audit

## Status

Complete ✅ (2026-05-14)

## Story

**As a** sales user,  
**I want** messages, tasks, timeline, documents/archive, and audit log to reflect real deal activity,  
**so that** every commercial interaction is traceable from one deal workspace.

## Acceptance Criteria

1. Deal timeline is append-only and records system, user, and cross-module events.
2. Messages support creation, listing, role/author metadata, and mention/autocomplete behavior where currently exposed.
3. Tasks support creation, assignment, due date, status transition, and deal linkage.
4. Documents/archive behavior is implemented or explicitly blocked by source docs with a concrete decision.
5. Audit log and audit export include deal, actor, event type, timestamp, and metadata.
6. Tests cover message, task, timeline append-only, audit export, and archive/document behavior.
