## Story

**As a** commercial operations owner,  
**I want** FBR-Click reviewed and corrected against its original CRM concept,  
**so that** SQL intake, pre-sales workflow, human/agent collaboration, and downstream commercial handoff fully match the intended product.

## Acceptance Criteria

1. The implemented Click scope is reviewed against `fbr-portal-docs/02_CLICK/PRD.md`, `fbr-portal-docs/02_CLICK/SPEC_TASKLIST.md`, `fbr-portal-docs/02_CLICK/TASKS_CLICK.md`, and `fbr-portal-docs/02_CLICK/concept.html`, with all shipped capabilities classified as conformant, partial, or missing.
2. Core CRM functionality matches the original concept: 5-stage pipeline, automatic SQL intake from FBR-Leads, manual deal creation, archive/discard flow with reason, deal details, timeline, tasks, notes, documents, and KPIs.
3. Human/agent pre-sales behavior matches the original concept: Comercial Bot, Discovery Bot, Proposta Bot, Follow-up Bot, Briefing Bot, and Handoff Bot operate with visible status and immutable audit evidence where documented.
4. Information flow matches the original ecosystem design: `lead.qualified` enters from FBR-Leads, internal notifications and messages stay inside Click hub behavior, and `deal.closed` exits toward FBR-Sales without cross-module database shortcuts.
5. Audit log, role boundaries, and proxy/security behavior match the original docs and the portal-wide general architecture.
6. All identified Click gaps produce concrete code and test changes without adding new CRM behaviors outside the original docs.
7. Quality gates pass for the final conformity increment: `npm run lint`, `npm run typecheck`, `npm test`, and `npm run build`.
