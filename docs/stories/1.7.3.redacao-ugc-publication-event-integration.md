## Story

**As a** newsroom operator,
**I want** Eu Reporter UGC, WordPress publication, alert classification, and `article.published` integration wired through backend contracts,
**so that** community submissions and published articles can flow into newsroom and FBR-Sales operations with human control points.

## Acceptance Criteria

1. UGC public submission accepts `{ submissor, email, cidade, descricao }` and validates minimum description length, email, covered city, forbidden words, and suspicious links.
2. UGC confidence score follows the parent story scoring rules and sets `auto_aprovavel=true` only for score `>= 60`.
3. Accepting UGC creates an article with `etapa='coletado'` and triggers the editorial pipeline.
4. Credit rules apply: +10 when a pauta is accepted, +20 when the derived article is published, and each 100 credits gives community highlight eligibility.
5. WordPress publication uses `WP_API_URL`, `WP_USER`, and `WP_APP_PASSWORD`; sends title, content, status, categories, optional featured media, and meta fields for `cidade`, `tipo_conteudo`, and `fonte_original`.
6. WordPress 5xx errors retry 3 times and then move article to `etapa='erro'`.
7. Alert classification supports all 8 parent-story alert types with the listed default levels, automatic actions, and auto-resolution rules.
8. `article.published` event payload matches the parent-story TypeScript contract and updates FBR-Sales audience/media kit data.
9. Tests cover UGC accept/reject, duplicate/invalid input, scoring/credits, alert classification/resolution, WordPress payload/retry, notification webhook, and event payloads.
10. Repository-level gates pass: `npm run lint`, `npm run typecheck`, and `npm test`.

### UGC / Eu Reporter Rules

- Public submission payload: `{ submissor, email, cidade, descricao }`.
- Initial validation: description has at least 20 characters, email is valid, city is covered, and spam filters run against forbidden words/suspicious links.
- Confidence score is 0-100: +30 registered email, +20 covered city match, +25 description over 100 characters and coherent, +15 prior accepted submissions, -40 external links, -30 spam words.
- Score `>= 60` sets `auto_aprovavel=true`; lower scores remain pending manual review with `auto_aprovavel=false`.
- Accepting UGC creates an article with `etapa='coletado'` and triggers the editorial pipeline.
- Credits: +10 when a pauta is accepted, +20 when the derived article is published, and every 100 credits gives highlight on the "Reporteres da Comunidade" page.

### Alert Classification

| Tipo | Nivel padrao | Automatic action |
|---|---|---|
| `falha_agente` | `error` | Notify operator via SSE and move article to dead letter. |
| `fonte_indisponivel` | `warn` | Pause source automatically and notify operator. |
| `conteudo_sensivel` | `warn` | Block automatic publication and require human review. |
| `queda_performance` | `warn` | Notify operator and suggest checking sources. |
| `qualidade_baixa` | `info` | Requeue Editor review. |
| `imagem_nao_encontrada` | `info` | Use placeholder image and mark article for media review. |
| `limite_api_atingido` | `error` | Pause calls for 1 hour and schedule automatic retry. |
| `ugc_spam_detectado` | `info` | Mark UGC submission as rejected and do not notify operator. |

Auto-resolution rules:
- `fonte_indisponivel`: auto-resolved when the feed returns HTTP 200 OK again.
- `limite_api_atingido`: auto-resolved after 1 hour.
- `falha_agente`: auto-resolved when the article is processed successfully in retry.

### WordPress Integration

- Authentication uses WordPress Application Passwords or JWT plugin.
- Environment variables named by source docs: `WP_API_URL`, `WP_USER`, `WP_APP_PASSWORD`.
- Post payload includes `title`, `content`, `status` (`publish`, `draft`, or `pending`), `categories`, optional `featured_media`, and meta fields for `cidade`, `tipo_conteudo`, and `fonte_original`.
- WordPress 5xx errors retry 3 times; if still failing, article becomes `etapa='erro'`.

### Article Published Event

```typescript
interface ArticlePublishedEvent {
  event: 'article.published';
  data: {
    artigo_id: string;
    titulo: string;
    cidade: string;
    canal: string;
    url: string;
    data_publicacao: string;
    audiencia_estimada?: number;
  };
}
```

FBR-Sales action: update audience data for the corresponding media kit.
