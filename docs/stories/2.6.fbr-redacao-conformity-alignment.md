## Story

**As a** newsroom operations owner,  
**I want** the current portal-scoped Redacao implementation reviewed and aligned against the original product docs,  
**so that** the shipped module stops pretending to be complete and becomes measurably conformant.

## Acceptance Criteria

1. The current implementation in `apps/portal/src/app/redacao/*`, `apps/portal/src/app/api/proxy/redacao/*`, and `apps/portal/src/lib/redacao/*` is compared against:
   - `fbr-portal-docs/06_REDACAO/PRD_SPEC_TASKLIST.md`
   - `fbr-portal-docs/06_REDACAO/TASKS_REDACAO.md`
   - `fbr-portal-docs/06_REDACAO/concept.html`
   - `fbr-portal-docs/00_GERAL/DOCUMENTO_GERAL.md`
2. The comparison explicitly classifies each major Redacao capability as:
   - conformant
   - partial
   - missing
3. The story closes documentation drift from stories `1.7`, `1.7.1`, `1.7.2`, `1.7.3`, and `1.7.4`.
4. The resulting implementation plan does not invent scope beyond the source docs.
5. Final delivery for this conformity increment includes updated code, tests, checklist state, file list, and passing `npm run lint`, `npm run typecheck`, `npm test`, and `npm run build`.
