# Story 3.4.2: FBR-Redacao Editorial Pipeline and WordPress Publishing

## Status

Complete ✅ (2026-05-14)

## Story

**As an** editorial lead,  
**I want** content to move through Monitor, Jornalista, Midia, Editor, and Publisher until WordPress publication,  
**so that** Redacao delivers the automated newsroom workflow described in the PRD.

## Acceptance Criteria

1. Editorial pipeline stages match the PRD agents: Monitor, Jornalista, Midia, Editor, Publisher.
2. Articles move through validated stage transitions with timestamps and actor/agent metadata.
3. Media upload and editorial category mapping are implemented.
4. WordPress REST API publishing path is implemented behind approved configuration.
5. Published content appears in `Publicados`.
6. Retry/failure states are observable for external publishing failures.
7. Tests cover stage transitions, media mapping, WordPress publish success/failure, and published listing.

## Tasks

- [ ] Define pipeline state machine from PRD/SPEC.

- [ ] Implement media upload/category mapping contracts.
- [ ] Implement WordPress client/proxy with safe configuration.
- [ ] Add retry/failure handling for publish jobs.
