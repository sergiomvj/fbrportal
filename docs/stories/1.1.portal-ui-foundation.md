## Story

**As a** FBR Portal operator,
**I want** the Portal UI monorepo, session package, portal bridge package, environment validation, auth routes, route middleware, and login page,
**so that** every later portal screen and module embedding flow starts from a secure authenticated foundation.

## Acceptance Criteria

1. The monorepo initializes with `apps/portal`, `packages/ui`, `packages/auth`, `packages/portal-bridge`, and `packages/tsconfig`, with TypeScript strict, shared ESLint/Prettier, and Turborepo pipeline `build -> lint -> test`. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P01`]
2. `packages/auth` exports `getSession()`, `withSession()`, `destroySession()`, `SessionData { userId, email, role, empresaId }`, `useSession()`, and iron-session config using cookie `fbr_session`. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P02`]
3. `packages/portal-bridge` exports `PortalMessage`, `sendPortalMessage()`, `usePortalBridge()`, and `isValidPortalMessage()` for the 5 variants `AUTH_TOKEN`, `NAVIGATE`, `NOTIFICATION`, `MODULE_READY`, and `CROSS_MODULE_EVENT`. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P03`]
4. Environment validation exists via `.env.example`, `apps/portal/src/env.ts`, and server-only env access, with `SESSION_SECRET` minimum length of 32 chars and valid Supabase URLs. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P04`]
5. `/api/auth/login` validates input, rate-limits to 5 requests/min per IP, creates an iron-session for valid credentials, returns generic errors, and never exposes Supabase tokens, stack traces, or internal details. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P05`]
6. `/api/auth/logout` is idempotent and destroys the local session; `/api/auth/me` returns sanitized session user data and never exposes the Supabase Auth UUID. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P06`]
7. `apps/portal/src/middleware.ts` protects all routes except `/login`, `/api/auth/*`, and static assets; unauthenticated protected routes redirect to `/login` while preserving callback URL. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P07`]
8. `/login` renders an accessible dark FBR login form with Zod/React Hook Form validation, loading state, generic invalid-credential error, Enter-submit behavior, and redirect on success. [Source: `fbr-portal-docs/01_PORTAL_UI/TASKLIST.md#TASK-P08`]
9. Quality gates pass for this increment: lint, typecheck, tests, and build if available.
