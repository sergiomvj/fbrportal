# Story 3.4.1: FBR-Redacao Data, RSS, and UGC Intake

## Status

Complete ✅ (2026-05-14)

## Story

**As an** editorial operator,  
**I want** RSS sources and UGC submissions persisted and moderated,  
**so that** Redacao starts from real incoming content instead of mocked newsroom state.

## Acceptance Criteria

1. RSS sources support CRUD, activation, pause, and company/editorial scoping.
2. RSS collection stores source item metadata and ingestion status.
3. UGC queue persists submissions, moderation state, reporter metadata, and rejection/acceptance reasons.
4. Accepted RSS/UGC items enter the editorial pipeline.
5. Mock-only store behavior is removed from execution paths covered by this story.
6. Tests cover RSS CRUD, activation/pause, UGC accept/reject, and pipeline entry.
