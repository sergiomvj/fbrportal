## Story

**As a** FBR Portal user,
**I want** a shared visual system and Arva agent picker available to every module,
**so that** modules share consistent UI patterns and can connect managed agents through the same integration contract.

## Acceptance Criteria

1. Shared CSS variables, grid background, glow effects, fonts, animations, glass nav, card patterns, KPI grid, and hero pattern are implemented from `TASKLIST_GERAL.md`.
2. `AgentPicker` exists and supports listing Arva agents, resolving an agent identity, filtering by tags, linking agents to a module, and displaying linked agents with heartbeat state.
3. `packages/arva-integration` exists with typed functions `listAgents(companyId)`, `resolveAgent(fbrchatId)`, and `openChat(agentId, userId)`.
4. Required environment variables are documented: `ARVA_BASE_URL` and `ARVA_FBRCHAT_SHARED_TOKEN`.
5. Each module can consume the shared patterns without duplicating CSS tokens or agent API client logic.
6. The story does not implement module-specific business workflows; it only creates reusable design and integration primitives.
