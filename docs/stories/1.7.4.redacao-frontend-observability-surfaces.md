## Story

**As a** newsroom operator,  
**I want** Redacao pages for overview, production, published articles, sources, UGC, alerts, and agents,  
**so that** I can inspect the mocked newsroom state from within the portal.

## Acceptance Criteria

1. The story reflects the current frontend footprint that already exists under `apps/portal/src/app/redacao/*`.
2. The story does not claim live SSE, real operator controls, or source-doc parity where the UI is still static or store-backed.
3. Open UI gaps versus the source docs are explicit.

## Tasks / Subtasks

- [x] Document the existing Redacao page set and shared shell. (AC: 1)
- [x] Document the current component set used by the portal prototype. (AC: 1)
- [x] Remove incorrect claims of full observability parity. (AC: 2)
- [ ] Add missing source-doc behaviors such as richer controls, live updates, and stronger intervention flows. (AC: 3)
- [ ] Add Redacao-focused frontend automated tests. (AC: 3)
